# fkmyjcs_miners

> Minecraft 1.20.1 (Forge 47.4.21) 模组 —— 以「矿脉」为核心的探矿与多方块采矿体系（开发中）。

[![License: CC BY-NC 4.0](https://img.shields.io/badge/License-CC%20BY--NC%204.0-blue.svg)](LICENSE)

## 当前状态（重要）

本模组目前处于**框架 / 原型阶段**。已经做好的是"数据、信息显示与结构骨架"类基础设施：

- 矿脉的数据模型，以及按坐标查询 / 修改矿脉的能力；
- 探矿工具，以及手持时右侧常驻的矿脉信息显示；
- 多方块结构的预览与自动建造。

但**尚未实现任何"产出矿物"的可玩功能**。下面列出的「按矿脉挖掘、虚空采矿、地心采矿、多种矿机、多样化插件升级」全部处于**规划**状态，没有落地；多方块自动建造也只是"按图纸把方块摆出来"，并不生产矿石。

> 换句话说：**当前版本没有实际可玩的生产玩法**，它更像一套为将来采矿玩法准备的地基。

## 玩法内容

### 矿脉系统

游戏里每一块区块（16×16 的一片区域）都会生成一条"矿脉"，记录这片地下藏着什么矿、比例多少、总共还剩多少（储量）。一条矿脉包含：

- **主矿**：这条矿脉的主要矿物；
- **次矿**（可能为空）：次要矿物；
- **岩石**：填充用的普通石头类方块；
- **权重**：主矿 / 次矿 / 岩石各自的出现比例，三者之和恒为 100；
- **储量**：这条矿脉还能被开采的总量。

矿脉分三种类型：**单矿**（只有主矿）、**双矿**（主矿 + 次矿）、**贫矿**（矿物很少，绝大部分是岩石）。这些数据会随存档持久化保存，一旦生成就不会被以后更新的代码改掉。除了自动生成，模组还支持**按坐标查询任意一处矿脉的详情**，以及**按坐标修改**某处矿脉的类型、矿物、权重或储量。

### 矿脉生成逻辑

矿脉是**按区块确定性生成**的，也就是说：同一个区块永远得到同一条矿脉，不随重新进入世界而改变。

- **随机种子**：由"区块坐标 + 世界维度 + 一个生成盐值"共同算出（`种子 = 区块键 ^ 固定混淆常数 ^ 盐值`）。因为种子里**不含版本号**，所以已经保存到存档里的矿脉永远不会被新版本代码覆盖。
- **类型概率**：单矿约 20%、双矿约 60%、贫矿约 20%。
- **权重分布**：单矿的主矿权重约 65（上下浮动 10）；双矿的主矿约 45、次矿约 20（均上下浮动 10）；贫矿的主矿约 30；纯岩石矿脉三种权重为 100% 岩石。无论怎么浮动，三者之和始终被归一到 100。
- **初始储量**：由配置项的基准值上下浮动得到（默认基准 200000、浮动范围 50000）。
- **持久化**：矿脉信息按维度保存在存档文件里；在单人 / 客户端一侧只会临时算出矿脉用于显示，不会写回存档。

### 探矿工具

- **探矿杖**：有使用次数（耐久），右键可以探明脚下这块区块有什么矿。
- **探矿仪 / 高级探矿仪**：靠电力（FE/RF）驱动，消耗电量来探测区块；高级款还能打开一张 7×7 的矿脉地图。
- 手持这类工具时，屏幕右侧会常驻显示**当前所在区块的矿脉信息**（主矿 / 次矿 / 岩石及各自概率、是否已探明），可用快捷键开关显示。

### 多方块结构

可以用多个方块按图纸拼出一台"机器"（例如金字塔状的大型结构）。结构支持：

- **带 NBT 的方块**：图纸里某个位置可以指定方块的附加数据（如箱子内容），并可选择是否在自动建造时把这部分数据写进世界；
- **候选组**：某个位置可以放"一组可互换方块中的任意一个"，只要世界里的方块落在组内任一个就算成形；
- **方向绑定**：默认结构只认控制器当前朝向；也可设为"不绑定方向"，此时校验会遍历控制器的四个水平朝向，任一能成形即视为结构成立；
- **自动建造**：手持指定物品右键控制器，即可自动按图纸搭建——创造模式瞬间完成，生存模式按每秒方块数消耗背包材料逐格建造。

控制器方块的物品提示栏会显示该结构的包围盒尺寸（宽 × 高 × 深）。

### 物品管理器集成

在 **JEI / REI / EMI** 这三个物品管理器里都能**分层预览多方块结构**（每层示意图、材料清单、结构名称与 id）；配合 **Jade**，看向控制器方块时会显示"已成形 / 未成形"；还支持在 **AE2** 的样板编码终端里一键把结构材料导出成处理样板。

### 规划中的功能（尚未实现）

- **按矿脉挖掘**：让矿机读取所在区块的矿脉，消耗矿脉储量来产出对应矿物。
- **虚空采矿**：不依赖具体矿脉，按概率或配置表持续产出矿物。
- **地心采矿**：面向后期的高产出、高能耗玩法。
- **多种矿机**：对应上述不同采矿方式的各种机器机型。
- **多样化插件升级**：给矿机插入升级件以改变速度、产量、能耗、范围等。

## 自定义拓展

本模组为整合包作者与模组开发者预留了多处自定义空间，可通过**配置、脚本、数据包**三种方式自由调整，无需改动 Java 代码：

### 配置文件（Forge Config）
提供服务端配置文件，可直接改写模组的平衡性与可用维度：
- **自动建造**：触发物品、创造模式建造耗时、生存模式每秒方块数、大结构目标耗时倍数；
- **探矿**：探矿仪与高级探矿仪的电力容量与单次耗电；
- **矿脉**：储量基准与浮动范围、各维度（主世界 / 下界 / 末地）矿石池、末地是否生成矿物。

### KubeJS / 编程接口
模组暴露了"按坐标查询 / 修改矿脉"的编程接口，KubeJS 脚本或其它模组可直接调用：按区块坐标读取矿脉的主矿 / 次矿 / 岩石、各自权重与储量，或改写其类型、矿物、权重、储量，甚至整条重新随机生成。同时附带 `/vein` 指令，便于在游戏内或脚本中调试与批量操作。

### 数据包（多方块结构）
多方块结构以 JSON 数据包形式提供（`data/<模组id>/multiblock/*.json`）。你可以通过数据包**新增自己的机器结构，或覆盖已有结构**，并享有充分的自由度：
- **候选组**：某个位置放"一组可互换方块中的任意一个"；
- **方块 NBT**：指定方块附加数据，并可选择自动建造时是否写入世界；
- **方向绑定**：结构可设为只认控制器朝向，或遍历四个水平朝向、任一成形即成立；
- **分层图纸**：用字符矩阵描述每一层，控制器层置于顶层。

通过上述组合，整合包可以定义出形态、材料、朝向规则各异的机器，而无需编写代码。

## 构建

```bash
# 克隆后，使用 Gradle 包装器构建
./gradlew build -x test
# 产物位于 build/libs/*.jar
```

> 首次构建会下载 Minecraft 与依赖并解映射，耗时数分钟。需要联网。

## 可选集成（软依赖，缺失仍可运行）

以下模组为**可选**集成，缺失时本模组仍可正常运行：

- **JEI** (15.20.0+) · **REI** (12.1.0+) · **EMI** (1.1.0+) —— 多方块结构预览
- **Jade** (11.0.0+) —— 看向控制器显示「已成形 / 未成形」
- **AE2** (15.0.0+) —— JEI 样板编辑器「+」键一键导出多方块材料
